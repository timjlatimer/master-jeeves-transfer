## 8. THE CONSTITUTIONAL MEMORY SYSTEM — COMPLETE SPECIFICATION

The Constitutional Memory System is a 4-layer defense-in-depth architecture designed in direct response to the KEI/ElevenLabs trust-breaking incident. It ensures standing directives, user preferences, institutional knowledge, and decision history are never lost, drifted from, or overridden by subtasks.

**Infrastructure cost:** $0 (GitHub + Markdown + existing Manus platform).

**Layer 1 — The Fireproof Safe (Storage):** Six-file hierarchy in a GitHub repository. Authority levels from SUPREME (Constitution — Tim only) through CURRENT (Active Context — rebuilt per session). Version-controlled, human-readable, AI-compatible, portable, free.

**Layer 2 — The Village Crier (Injection):** Before ANY subtask executes, it receives a mandatory Directive Injection Package (~1,200 tokens). Constitutional Principles always included in full, never filtered. Conservative inclusion policy — when in doubt, include. Self-reinforcing: the injection directive is itself stored in CONSTITUTION.md.

**Layer 3 — The Checkpoint (Verification):** Three-step gate: Extract relevant directives, Scan output against directives, Classify as COMPLIANT/FLAGGED/VIOLATION. Fresh Eyes verification — gate receives only directives + output, not the subtask's reasoning. Prevents "marking your own homework."

**Layer 4 — The Watchtower (Drift Detection):** Five triggers: session start, pre-subtask, post-subtask, context compression, weekly Monday audit. NEVER-COMPRESS safeguard ensures constitutional directives survive context window compaction verbatim. Weekly audit: 2-minute read for Tim, automated by Master Jeeves.

**Tim's ongoing time investment:** ~5 minutes per week (confirm/correct directive captures as needed + glance at weekly audit).
