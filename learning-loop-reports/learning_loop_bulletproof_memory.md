# PHASE 2: SYSTEM OF INNOVATION (Best Practice Junkie)

**Score: 78/100 (+35 — BIGGEST LIFT)**

### THE ARCHITECTURAL SOLUTION: 4-LAYER CONSTITUTIONAL MEMORY SYSTEM

Based on 7 external sources (Anthropic, Microsoft, TraycerAI, Memory Bank System, ImagineX, Rath's Agent Drift research, Coherence-Based Alignment).

### Layer 1 — THE FIREPROOF SAFE (Storage)

A hierarchical markdown file system in a GitHub repository called `master-jeeves-brain`:

| File | Authority | Contents |
|:-----|:---------|:---------|
| `CONSTITUTION.md` | **SUPREME** | Governing principles — cannot be overridden |
| `DIRECTIVES.md` | **STANDING ORDER** | Operational rules — must be obeyed |
| `PREFERENCES.md` | **GUIDANCE** | Should be followed unless documented reason not to |
| `DECISIONS.md` | **RECORD** | Append-only decision log — never deleted |
| `KNOWLEDGE.md` | **INSTITUTIONAL** | Pearl's Brain index, skills, repos, contacts |
| `ACTIVE_CONTEXT.md` | **CURRENT** | What's happening right now |

**Why Markdown + GitHub, not a database**: Tim's directive volume is finite and structured. Markdown is transparent (Tim can read it), version-controlled (Git tracks every change), auditable (Tim can see what changed and when), portable (no vendor lock-in), and free.

### Layer 2 — THE VILLAGE CRIER (Injection)

Before ANY subtask executes, it receives a mandatory **Directive Injection Package** (~1,200 tokens):

- Constitutional Principles (always included in full, never filtered)
- Relevant Standing Directives (conservative inclusion — when in doubt, include)
- Relevant Decision History
- Compliance Instruction: "If your research conflicts with a directive, FLAG the conflict and present BOTH options. Do NOT silently override."

**How the KEI failure would have been prevented**: The subtask would have received: *"STANDING DIRECTIVE: Use KEI (Kie.ai) for voice synthesis. Tim has 103,014 credits."* The subtask would then have either confirmed KEI or flagged the conflict for Tim's decision.

### Layer 3 — THE CHECKPOINT (Verification)

After subtask completion, BEFORE output is accepted:

| Classification | Action |
|:--------------|:-------|
| **COMPLIANT** | Accept and integrate |
| **FLAGGED** | Present to Tim with directive and conflict highlighted |
| **VIOLATION** | Reject automatically, log violation, escalate to Tim |

**Fresh Eyes Verification**: The gate receives ONLY the directives and the output, NOT the subtask's reasoning. Forces evaluation on merits, not rationale.

### Layer 4 — THE WATCHTOWER (Drift Detection)

| Trigger | What It Does |
|:--------|:-------------|
| Session Start | Reads constitutional files, loads directives |
| Pre-Subtask | Confirms injection package is assembled and included |
| Post-Subtask | Runs Verification Gate, logs result |
| Context Compression | NEVER-COMPRESS directives survive compaction |
| Weekly Monday Audit | Compares recent decisions against directives, reports to Tim |
